A Option A — Run with npm start Recommended

The fastest way to run the project with zero extra configuration.

1 Open a terminal in the project folder

cd C:\Vardhan-Nvidia\edsprojectsetup

2 Install dependencies

npm install
What this installs serve (static file server) and other dev tools from package.json. Only needed once.

3 Start the server

npm start

This runs npx serve . --listen 3001 and serves the project at http://localhost:3001.

4 Open in browser

Navigate to http://localhost:3001 in Chrome or Edge. You should see:

  • Sticky header with NVIDIA logo and navigation
  • Hero section — "Contact NVIDIA / Get Your Questions Answered."
  • Three contact cards (Support, Enterprise Sales, Find a Partner)
  • Regional offices with Americas / Asia / Europe tabs
  • Footer with link columns and social icons

Stop the server

Press Ctrl + C in the terminal, or run this from a second terminal window:

npm stop
How npm stop works Runs npx kill-port 3001 — finds and kills whatever process is bound to port 3001. Useful when npm start is running in the background, inside an IDE task runner, or in a terminal you can't reach.

Why port 3001? The AEM CLI (aem up) reserves port 3000 by default. Using 3001 for the static server lets both run side by side without file-serving conflicts.

B Option B — Run with the AEM CLI

The official Adobe tool for EDS development. Provides hot reload and can proxy content from a live AEM backend.

1 Install the AEM CLI globally

npm install -g @adobe/aem-cli

2 Start the dev server

cd C:\Vardhan-Nvidia\edsprojectsetup
aem up

Server starts at http://localhost:3000 (AEM CLI default port) with:

  • Hot reload — changes to .css or .js files update the browser instantly
  • Content proxy — fetches page content from the configured AEM origin

3 Verify hot reload

  1. Open blocks/hero/hero.css in your editor
  2. Change the background gradient value
  3. Save — the browser updates without a full reload

Stop the server

Press Ctrl + C in the terminal, or from a second terminal window:

npm stop

C Option C — Run with Python

No Node.js install required. Python 3's built-in server handles ES module MIME types correctly.

cd C:\Vardhan-Nvidia\edsprojectsetup
python -m http.server 3001

Open http://localhost:3001.

Important Never open index.html directly via file:// — ES modules require a proper HTTP server with correct MIME types.

Deploying to a Real EDS Project

Connect this POC to a live Adobe Edge Delivery Services project in 5 steps.

1 Create a GitHub repo from the AEM boilerplate

  1. Go to github.com/adobe/aem-boilerplate
  2. Click "Use this template" → "Create a new repository"
  3. Name it (e.g., nvidia-contact-eds) and set visibility to Public

2 Install the AEM Code Sync GitHub App

  1. Go to github.com/apps/aem-code-sync/installations/new
  2. Select "Only select repositories" and choose your new repo
  3. Click Install

3 Copy block files into the new repo

blocks/hero/
blocks/cards/
blocks/contact-info/
blocks/header/         (replace the boilerplate default)
blocks/footer/         (replace the boilerplate default)
styles/styles.css      (replace the boilerplate default)
index.html             (your contact page)

4 Author content in Google Docs / SharePoint

In a real EDS project, page content comes from a document. The table structure maps to HTML blocks:

Document elementEDS HTML output
Heading 1<h1>
Paragraph<p>
Table with "Cards" in first cell<div class="cards block">
Horizontal rule ---New <div class="section">

5 Preview and publish

After pushing code to GitHub, your site is immediately live at:

https://<branch>--<repo>--<owner>.aem.page/

Install the AEM Sidekick Chrome extension to preview and publish individual pages from Google Docs.

How EDS Works — Quick Reference

Browser loads index.html │ ▼ scripts.js runs (type="module") │ ├─► loadEager() │ └─ decorateMain(main) │ ├─ decorateSections() → adds .section, hides all sections │ └─ decorateBlocks() → adds .block + data-block-name │ ├─► loadLazy() │ ├─ loadBlocks(main) │ │ └─ for each .block: │ │ ├─ loads /blocks/<name>/<name>.css │ │ ├─ imports /blocks/<name>/<name>.js │ │ ├─ calls decorate(block) │ │ └─ reveals the section │ ├─ loads header block │ ├─ loads footer block │ └─ loads lazy-styles.css │ └─► loadDelayed() └─ analytics, chat widgets, non-critical scripts

Block anatomy

Every block is a self-contained folder with exactly two files:

blocks/my-block/
├── my-block.css    ← scoped styles   (.my-block { ... })
└── my-block.js     ← export default function decorate(block) { ... }

The decorate(block) function receives the raw block DOM element — its children are the table rows from the authored document. The function reshapes the HTML into any structure needed.

Troubleshooting

Page is blank / entirely black
The body starts at opacity: 0 and only becomes visible after scripts.js adds the appear class. If it stays black:
  • Open DevTools (F12) → Console tab and look for errors
  • Make sure you opened the page via http://localhost:3001, not by double-clicking index.html (file:// URLs block ES modules)
  • Confirm the dev server is still running in your terminal
"Failed to load module" errors in console
The block JS file path must exactly match the block's CSS class name. A <div class="contact-info"> needs: 
blocks/contact-info/contact-info.js
blocks/contact-info/contact-info.css
Port 3001 is already in use
Use a different port: 
npx serve . --listen 3002
Then open http://localhost:3002.

Note: port 3000 is reserved by the AEM CLI (aem up). If you need to stop whatever is on 3001, run npx kill-port 3001.
npm command not found
Install Node.js (LTS version) from nodejs.org. npm is bundled with it. Restart your terminal after installing.
Changes not appearing in the browser
  • With npm start / npx serve: hard-refresh with Ctrl + Shift + R
  • With aem up: .css and .js changes hot-reload automatically; .html changes need a manual refresh (F5)
